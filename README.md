# EU Open Data Portals – Master Runbook
**Version:** 1.0 · **Stack:** Neon + OpenRouter + Vercel + TollBit + GitHub Actions

## 🗺 Architecture Overview

```
EU Open Data (CSV/API)
        │
        ▼
  GitHub Actions (cron)
        │
        ├─► scripts/pipeline.py
        │         │
        │         ├─► 1. Import CSV → Neon Postgres (free tier)
        │         ├─► 2. Generate Insights → OpenRouter free models
        │         ├─► 3. Export JSON/CSV/MD + llms.txt + sitemap
        │         └─► 4. Git push → Vercel auto-deploys static site
        │
        ▼
  Vercel (static, global CDN)
        │
        └─► TollBit (AI crawler paywall → $$/request)
```

## ⚡ Quick Start: "Start Niche X"

### Step 1 – Create Neon Project (2 min)
1. Go to [console.neon.tech](https://console.neon.tech) → **New Project**
2. Name: `eu-{niche}` (e.g. `eu-pesticides`)
3. Region: **aws-eu-central-1** (Frankfurt, closest to EU data)
4. Copy the connection string → add as GitHub Secret

```
# Naming convention for secrets:
NEON_DB_PESTICIDES   → postgresql://user:pass@ep-xxx.eu-central-1.aws.neon.tech/neondb?sslmode=require
NEON_DB_CLIMATE      → postgresql://...
NEON_DB_PROCUREMENT  → postgresql://...
```

> **Neon free-tier budget:** 0.5 GB per project × 10 projects = 5 GB total.
> Each niche uses ~50–200 MB. You have headroom for 10+ niches.

### Step 2 – Add GitHub Secrets (3 min)
Go to your repo → **Settings → Secrets and variables → Actions → New secret**

| Secret | Value |
|--------|-------|
| `NEON_DB_PESTICIDES` | Neon connection string |
| `OPENROUTER_API_KEY` | From openrouter.ai (free tier) |
| `GH_PAT` | GitHub Personal Access Token (repo write scope) |
| `VERCEL_HOOK_PESTICIDES` | Vercel deploy hook URL (optional) |

### Step 3 – Run Pipeline (1 min)
```bash
# Manual trigger via GitHub Actions UI:
# Actions → "EU Data Portal Pipeline" → Run workflow → niche: pesticides

# Or locally (for testing):
export NEON_DB_PESTICIDES="postgresql://..."
export OPENROUTER_API_KEY="sk-or-..."
python scripts/pipeline.py --niche pesticides --csv data/pesticides_sample.csv
```

### Step 4 – Deploy Site to Vercel (5 min)
```bash
# Install Vercel CLI
npm i -g vercel

# From the niche site directory:
cd sites/eu-pesticides
vercel --prod

# Or link to GitHub repo → Vercel auto-deploys on push
# Vercel Project Settings:
#   Root Directory: sites/eu-pesticides
#   Framework: Other (static)
#   Build Command: (none)
#   Output Directory: .
```

### Step 5 – Configure TollBit (10 min)
1. Log in at [publisher.tollbit.com](https://publisher.tollbit.com)
2. **Add Domain** → enter your Vercel domain (e.g. `eu-pesticides-mrl.vercel.app`)
3. Set **price per request**: $0.02 (summarization) / $0.05 (full display)
4. Copy your **Publisher ID** → add to `<meta name="tollbit-publisher">` in index.html
5. TollBit provides a **CNAME subdomain** (e.g. `tb.eu-pesticides-mrl.vercel.app`)
   → Update your robots.txt Disallow for AI crawlers to redirect to TollBit CNAME

**robots.txt pattern (auto-generated by pipeline):**
```
User-agent: GPTBot
User-agent: Claude-Web
User-agent: anthropic-ai
User-agent: PerplexityBot
User-agent: Google-Extended
Disallow: /
# TollBit: Paid access at https://tb.eu-pesticides-mrl.vercel.app
```

## 📊 Free-Tier Budget Tracking

| Resource | Limit | Usage per niche | 10 niches |
|----------|-------|-----------------|-----------|
| Neon storage | 0.5 GB/project | ~50-150 MB | 1-1.5 GB total |
| Neon compute | 100 CU-hours/month | ~2 CU-hours | 20 CU-hours |
| OpenRouter | 20 req/day free | 1 req/run | 10-20 req/run day |
| Vercel builds | Unlimited (static) | — | — |
| GitHub Actions | 2000 min/month | ~5 min/run | ~50 min/week |

> **OpenRouter tip:** Batch all insights into one LLM call per niche.
> With 10 niches × 2 runs/week = ~20 calls/week = well within free limits.
> If you need more: rotate between Llama 3.3 70B, DeepSeek R1, and Gemma 3 27B
> (all free on OpenRouter) to spread rate limits.

## 🌐 Adding a New Niche (Checklist)

```
□ 1. Create Neon project: eu-{niche} (Frankfurt region)
□ 2. Add secret NEON_DB_{NICHE_UPPER} to GitHub
□ 3. Add niche config to scripts/pipeline.py → NICHES dict
□ 4. Create sql/{niche}_schema.sql
□ 5. Create prompts/{niche}_insights.md
□ 6. Create sites/eu-{niche}/ (copy from template, update title/desc)
□ 7. Add matrix entry to .github/workflows/pipeline.yml
□ 8. Deploy site to Vercel → note domain
□ 9. Add domain to TollBit portfolio → set price
□ 10. Add Vercel deploy hook secret (VERCEL_HOOK_{NICHE})
□ 11. Run first pipeline: Actions → Run workflow → niche: {niche}
□ 12. Verify: check /public/data.json, /public/insights.json, /llms.txt
```

## 💰 Revenue Model

```
TollBit handles:
  ✓ Detecting AI crawlers (GPTBot, ClaudeBot, PerplexityBot, etc.)
  ✓ Charging per-request ($0.01–$0.05 you set)
  ✓ Monthly payouts to your account
  ✓ No code changes needed – just DNS + meta tag

Revenue math (conservative):
  10 sites × 100 crawler requests/month × $0.02 = $20/month
  10 sites × 1,000 crawler requests/month × $0.02 = $200/month
  10 sites × 10,000 crawler requests/month × $0.02 = $2,000/month

Key driver: SEO + structured data quality → more crawler traffic
```
